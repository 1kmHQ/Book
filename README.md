## Glossary of Key Terms

| Term | Definition |
|------|------------|
| **Zone** | The fundamental geographic unit of 1km operations — typically a taluk or cluster of villages |
| **Zone Admin** | A local person responsible for zone health: onboarding, support, governance |
| **Provider** | A driver/vehicle owner who offers transport services through the platform |
| **Prosperity Model** | Our approach where provider wellbeing is the foundation, not an afterthought |
| **Trust Loop** | The cycle where good service builds trust, which builds demand, which enables better service |
| **Slot** | A time block when a provider marks themselves available for bookings |
| **Scheduled Booking** | A ride booked in advance (hours or days) rather than on-demand |
| **Ride Group** | A bulk booking of multiple vehicles for coordinated logistics |
| **Business Account** | A verified business entity that can book, invoice, and manage team access |
| **Hyperlocal** | Operating at the smallest viable scale where trust and relationships are tangible |
| **Commission** | Platform's percentage of fare (2-5% in 1km, vs 20-25% in typical platforms) |
| **Instant Payout** | Provider ability to withdraw earnings immediately after ride completion |

## How This Book Evolves

This book is not frozen. It will evolve. Here's the process:

**Versioning**: The book is versioned. Each significant update gets a version number and date. Changes are documented in a changelog.

**Proposal**: Any team member can propose changes. Proposals must explain: what changes, why, and what's affected.

**Debate**: Significant changes require discussion. Not just among founders — zone admins, senior providers, and team leads should weigh in.

**Approval**: Principle changes require broad consensus. Tactical updates can be approved by founders.

**Documentation**: Every change is logged. Future readers should be able to trace how the book evolved.

**Communication**: When the book changes, everyone who uses it is notified.

The goal: evolution without erosion. The core stays stable. The details adapt.
